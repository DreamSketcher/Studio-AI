# ai_studio_core

Headless AI-студиоядро, извлечённое из [XTTS-Studio-AI](https://github.com/DreamSketcher/XTTS-Studio-AI).

Содержит только серверную/логическую часть — среду управления, TTS/RVC-пайплайны,
клиентов LLM (облачных и локальных), утилиты для текста/аудио и систему
обновлений/верификации. **GUI-слой на PySide6 сюда не входит** и будет
строиться поверх этого пакета отдельным этапом.

## Установка

Базовое ядро (без тяжёлых ML-зависимостей):

```bash
pip install -r requirements-core.txt
pip install -e .
```

С TTS (torch + Coqui TTS):

```bash
pip install -e ".[ml]"
```

С поддержкой локальных LLM (llama.cpp):

```bash
pip install -e ".[local-llm]"
```

С RVC:

```bash
pip install -e ".[rvc]"
```

## Архитектурные инварианты

- **Слоистая архитектура** (см. `.importlinter`): модули имеют право
  импортировать только нижележащие слои. `env_core` не знает о TTS/RVC/LLM/GUI.
  TTS/RVC могут использовать env_core и утилиты, но не наоборот.
- **Нет зависимости от фреймворков GUI.** В этом репозитории нет ни одного
  импорта `tkinter`, `customtkinter`, `PySide6`, `PyQt` и т.п.
  Проверяется в CI (`headless-tests` job) и статически (import-linter).
- **i18n-опционален.** `ai_studio_core.gpt_client` использует безопасный фолбэк,
  если модуль локализации не установлен в проекте-хосте.

## CI

- `Architecture gate` — import-linter и валидация манифестов плагинов.
- `Headless tests` — матрица ubuntu/windows/macos без GUI-зависимостей:
  импорт пакета в headless-режиме + прогон быстрых юнит-тестов.

Тесты, требующие `torch`/`TTS`/`rvc-python`/`llama-cpp-python`, будут добавлены
отдельными jobs по мере подключения соответствующих extras.

## Графический интерфейс (PySide6)

Поверх headless-ядра собран полноценный GUI на PySide6 в стиле оригинального
XTTS Studio (тёмная неоновая тема, боковая навигация, все основные рабочие области).

Запуск:

```bash
pip install -e ".[gui]"
python run_gui.py
```

Панели интерфейса:

| Иконка | Панель | Что делает |
|---|---|---|
| 🎙 | **TTS** | Основная область: ввод текста, параметры голоса, генерация, опции (prosody/RVC/chunker), прогресс |
| 💬 | **Чат** | Чат с облачными и локальными LLM-провайдерами, системный промпт |
| 📂 | **Пакетно** | Очередь из TXT-файлов/папок для массовой генерации с общими параметрами |
| 🎤 | **Голоса/RVC** | Библиотека референсных .wav, каталог RVC-моделей, доверие/поиск |
| 📖 | **Замены** | Словарь автозамен терминов перед озвучкой |
| ⭐ | **Пресеты** | Сохранение/загрузка/импорт/экспорт наборов параметров генерации |
| 🕑 | **История** | Журнал последних генераций |
| 🛠 | **Окружение** | Диагностика системы, статус зависимостей (torch/CUDA/…), установка компонентов |
| 🔄 | **Обновления** | Проверка обновлений и верификация подписей релизов |

Особенности:
- Все долгие операции (генерация, диагностика, установка) запускаются в
  отдельных `QThread` — UI не блокируется.
- Между UI и ядром есть безопасная прослойка `core_bridge`: интерфейс
  стартует даже при отсутствии опциональных ML-зависимостей и показывает
  их статус в сайдбаре и на вкладке «Окружение», вместо того чтобы падать
  с ImportError.
- Архитектурный гейт не распространяется на `ai_studio_gui/` (он находится
  ВЫШЕ всех слоёв в новой иерархии и может импортировать любые модули ядра).
