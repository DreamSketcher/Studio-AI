## 1. Что добавлено

### Уровень 1 — события → пресеты действий

| Компонент | Суть |
|---|---|
| `ui/experience/events.py` | Реестр из 10 событий (app_started, generation_complete/failed, queue_drained, project saved/loaded/new, download_finished, export_done, chat_reply). Опечатка в пресете → `ValueError` при загрузке, не молчание. |
| `ui/theme/presets/experience.default.json` | Mapping событие → `{toast, sound, accent_pulse}`. Toast только там, где приложение ещё не показывает свой (дедупликация). Toast-тексты: i18n-ключ или литерал с `{output}`-плейсхолдерами (безопасный format_map, без KeyError). |
| `ui/experience/sounds.py` | **Реальные** тона: numpy-синтез нотных последовательностей (start_soft, done_chime, error_low, success_soft, tick) в WAV 22050 Гц 16-bit mono с attack/release-огибающей; кэш `CACHE_DIR/experience/`. Никаких бинарных подделок. |
| `ui/experience/manager.py` | `ExperienceManager.handle(event, payload)` → toast/звук/пульс по пресету. Звук — `QSoundEffect`, guarded (без аудио-устройства тихо, статус в лог). Тумблер звуков отключает **только** звук (пульс/toast живут отдельно). |
| `AccentPulse` | Полоса 5 px поверх окна: вспышка → `QPropertyAnimation`-затухание → авто-скрытие. Никакого «перекрасили и забыли». |

### Уровень 2 — статистика → честная эвристика

| Компонент | Суть |
|---|---|
| `ui/experience/stats.py::UsageStats` | Счётчики из реальных событий: сессии, активации вкладок, действия (tts_generate/chat_send/export/download/project_save/…), использование бэкендов, активные секунды. Атомарный `json/usage_stats.json`. Неизвестные поля при загрузке игнорируются явно. |
| `suggested_start_workspace` | Детерминированная эвристика: ≥3 активаций → вкладка-максимум; ничья → дефолт tts; недостаточно данных → дефолт. Причина возвращается с решением и пишется в консоль (`exp_start_tab_reason`). |
| Проводка | Активации вкладок пишутся **только** при пользовательских переключениях (программный выбор эвристикой не засчитывается — данные не насилуются). Длительность сессии записывается в `closeEvent`. |
| Тумблеры | Settings → General: «Звуковые эффекты интерфейса» (`ui/exp_sounds`) и «Адаптивная стартовая вкладка» (`ui/adaptive_start_tab`) — обе в снапшоте `settings_changed`, обе персистятся. |

### Правило consent (заложено под уровень 3)

Transient-эффекты — без спроса; персистентное поведение — только через
явные опции пользователя. Ни одно событие не меняет интерфейс молча.

## 2. Тесты (новые, 22 шт., `test_experience_layer.py`)

- **Звуки:** каждый тон — реальный WAV (rate/channels/16-bit, размер,
  RMS анти-тишина, длительность = сумме нот ±1%); кэш не пересинтезирует
  (mtime стабилен); неизвестный тон — ValueError.
- **Пресеты:** дефолт валиден; неизвестное событие/действие/тон →
  точная ошибка; merge пользовательского пресета поверх базового.
- **Менеджер:** событие исполняет действия пресета; отключение звуков
  глушит play, но не пульс; resolve_text (i18n-ключ/литерал/безопасные
  плейсхолдеры); неизвестное событие → честный False, колбэки пусты.
- **AccentPulse:** показывается и **самоудаляется** за ~1 с (offscreen).
- **Статистика:** запись→диск→чтение roundtrip; отсутствующий файл — пусто
  без краша; мусорные поля отброшены; эвристика (threshold/winner/tie).
- **Интеграция (MainWindow):** seed chat×6+tts×1 → стартовая вкладка «Чат»;
  тумблер off → дефолт; без статистики → дефолт; тумблер звуков доезжает
  до менеджера и QSettings; переключение вкладки → активация на диске;
  queue_drained стреляет ровно один раз.

## 4. Осознанные границы

- Скрин-глазам 5 px пульс на 1440-окне малозаметен — программные ассерты
  (`pulse visible → hidden`) фиксируют поведение; толщину/цвет меняет
  пресет, не код.
- Оффлайн-окружение без аудио-устройства: звук реально синтезируется в файл
  (проверено побайтно), проигрывание — guarded no-op. На десктопе с
  аудио-выходом тот же путь звучит.
- «Пользовательские пресеты» читаются `merge_presets(base, path)` — UI-выбор
  пресета в Settings не добавлялся (уровень 1 задуман инфраструктурным).
