# Отчёт №7 — Краш переключения языка: причина, i18n EN/RU, живое переключение

**Дата:** 2026-07-18
**Этап:** Диагностика и устранение краша при переключении языка на русский;
полноценная система локализации (EN/RU) без заглушек.

---

## 1. Причина краша (воспроизведено и задокументировано)

Проверка показала три слоя проблемы:

| # | Что найдено | Последствие |
|---|---|---|
| 1 | `SettingsPanel.settings_changed = None` — поле класса-заглушка вместо сигнала | Любая попытка `emit()` (переключение комбобокса «Language:» → обработчик настроек) падает: `AttributeError: 'NoneType' object has no attribute 'emit'` — **это и есть краш, воспроизведён кодом** |
| 2 | Комбобокс языка не был ни к чему подключён, а ключей русского не существовало вообще | Выбор «Русский» невозможно было корректно обработать |
| 3 | `gpt_client` импортировал внешний модуль `i18n` (`from i18n import t`), которого в проекте нет — фолбэк возвращал сырые ключи | Подписи провайдеров: `prov_groq`, `prov_openrouter` вместо текста |

Воспроизведение (до правки):

```
AttributeError: 'NoneType' object has no attribute 'emit'
```

## 2. Что сделано

### 2.1 Новый модуль ядра `ai_studio_core/i18n.py`
- Реестр языков `LANGUAGES = {"en": "English", "ru": "Русский"}`.
- `set_language(code)` — неизвестный код безопасно отклоняется (`False`, язык не смещается) — закрыт класс ошибок «сломалось от неожиданного ввода».
- `t(key, lang=None)` — **никогда не бросает исключений**: нет перевода → английский; нет ключа → сам ключ.
- Полные словари EN/RU (**161 ключ × 2 языка**): меню, действия, вкладки, dock-заголовки, строки всех workspace и панелей, этапы пайплайна, статусные сообщения контроллеров, подписи провайдеров gpt_client (`prov_*`, `cat_*`).

### 2.2 Починен gpt_client
Импорт i18n теперь: `ai_studio_core.i18n` → внешний `i18n` → фолбэк. Существующая
`refresh_i18n_labels()` реально обновляет подписи провайдеров при смене языка.

### 2.3 Панель настроек — без заглушек
- `settings_changed = Signal(dict)`, `language_changed = Signal(str)`, `paths_changed = Signal(str, str)` — настоящие сигналы (краш-причина устранена).
- Комбобокс языка строится из `i18n.LANGUAGES` (только реально переведённые: English/Русский; 日本語/中文 убраны — переводов под них нет).
- Тема: только «Dark» (единственная реализованная; Light/System-заглушки убраны).
- Устройство: Auto/CPU; CUDA добавляется только если реально установлен torch и виден GPU.
- Кнопки путей открывают реальный выбор каталога (`QFileDialog`) и эмитят изменения.

### 2.4 Живое переключение языка (без перезапуска)
- `retranslate_ui()` у MainWindow (меню, вкладки, dock-заголовки) и у каждого
  workspace/панели (кнопки, группы, плейсхолдеры, pipeline-strip — добавлен
  метод `PipelineStrip.set_steps()`, `CollapsibleGroup.set_title()`,
  `FileDropZone.set_label()`).
- Выбранный язык персистится в `QSettings` (`ui/language`) и восстанавливается
  при старте (`app.py` применяет его до построения виджетов).
- Докам выставлен `objectName` (иначе `saveState()` спамил предупреждениями).

## 3. Тесты (новые, 26 шт.)

**`test/test_i18n_core.py`** (13):
- паритет ключей EN/RU (множества строго равны), все значения непустые;
- каждый ключ переводится обоими языками; русский ≠ ключ;
- `t()` не бросает на `None/""/число/пропавший ключ`; неизвестный язык → фолбэк;
- `set_language("de"/"日本語"/None/42/...)` → `False`, текущий язык не смещается (краш-регрессия);
- ping-pong EN↔RU ×10 стабилен;
- `gpt_client.refresh_i18n_labels()`: подписи провайдеров реально меняются ru↔en.

**`test/test_ui_language_switch.py`** (13, offscreen Qt):
- `settings_changed` — настоящий сигнал, `emit()` работает (краш-регрессия);
- комбобокс содержит только доступные языки; переключение эмитит код;
- полный цикл: вкладки/меню/dock-заголовки/kнопки/group-заголовки/pipeline-strip
  меняются на русский и обратно;
- 10 циклов переключения — ни одного исключения;
- выбор персистится в QSettings; `set_language("de")` игнорируется без последствий.
