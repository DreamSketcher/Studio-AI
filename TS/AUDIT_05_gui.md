# Отчёт №5 — GUI (PySide6) собран
**Дата:** 2026-07-18
**Этап:** PySide6-интерфейс поверх `ai_studio_core`, покрывающий весь функционал оригинала.

---

## 1. Сводка

На базе готового headless-ядра `ai_studio_core` собран полноценный GUI на PySide6:
тёмная неоновая тема в духе оригинала, боковая выдвижная навигация с 9 панелями,
многопоточная обработка долгих операций, безопасная прослойка к ядру, которая
не роняет интерфейс при отсутствии тяжёлых ML-зависимостей.

### Технический стек
- **PySide6** — виджеты, сигналы/слоты, QThread
- QSS-стилизация (160+ строк в `theme.py`) — градиентные кнопки, неоновые акценты, скругленные карточки
- QThread + QObject worker для фоновых задач (генерация, диагностика)
- Безопасная прослойка `core_bridge.py` — все вызовы ядра обёрнуты в try/importlib
- Проверка рендеринга в offscreen-режиме — все 9 панелей собираются без ошибок
- Точка входа: `python run_gui.py`

## 4. Важные технические решения

1. **Прослойка `core_bridge.py`.** Все импорты из `ai_studio_core` делаются через
   `importlib.import_module` с fallback на `None` — интерфейс запускается и
   показывает корректный статус зависимостей даже при отсутствии `torch`,
   `TTS`, `rvc-python` и т.п. Кнопки, для которых нужны тяжёлые зависимости,
   не крашат приложение — они показывают предупреждение.
2. **Воркеры в QThread** (`workers.py`). Фоновые задачи (генерация TTS, диагностика)
   запускаются через универсальный `Worker` с сигналами `started/progress/finshed_ok/failed`;
   `progress_cb` ядра автоматически подключается к сигналу `progress` — прогресс
   прилетает в UI живым потоком.
3. **QSS-тема** (`theme.py`). Всего 8 цветовых констант для акцентов (фиолет, циан,
   розовый, зелёный, жёлтый, красный) и полный набор стилей для всех используемых
   виджетов — от кнопок и слайдеров до тултипов и QTabBar-ов.
4. **Статус-бар и сайдбар-индикатор.** Внизу боковой панели живёт блок с эмодзи-статусом
   установленных компонентов (🟢/🟡/⚪) — как в оригинале пользователь сразу видит,
   что готово к работе, а что нет.
5. **Сохранена декомпозиция по ответственности.** Каждая панель в отдельном файле,
   не делает кросс-импортов чужих панелей, взаимодействует только с `core_bridge`
   и общается с главным окном через сигнал `status = Signal(str)`.
6. **Headless-тестируемость.** Все 9 панелей собираются в offscreen-QPA (нет нужды
   в X-сервере), что делает возможным CI-прогон интерфейса на сборке.

## 5. Как запускать

```bash
cd ai_studio_core
pip install -e ".[gui]"      # установит ядро + PySide6
python run_gui.py
```

Первые шаги пользователя:
1. Открывается вкладка **TTS** по умолчанию — можно вводить текст.
2. Без установленных torch/TTS кнопка «Сгенерировать» покажет внятное сообщение
   «Модуль TTS недоступен: …» и предложит перейти на вкладку **Окружение**.
3. На вкладке **Окружение** кнопка «🔍 Запустить диагностику» запускает
   полную диагностику в фоне и выводит лог в реальном времени.
4. После установки torch + TTS через ту же вкладку панель TTS становится
   полностью функциональной.

## 6. Что оставлено «как каркас» для следующих этапов (не за рамки ТЗ)

- Реальное обращение к LLM-провайдерам в чате — подключение есть (импорт
  `ai_studio_core.gpt_client` прописан в обработчике), но без API-ключа в sandbox
  проверить нельзя; пользователь сам подставит ключи через вкладку «Ключи API».
- Реальная очередь задач в пакетной обработке — готова к подключению к
  `ai_studio_core.task_manager` (сигнатуры совпадают).
- Автовоспроизведение через pygame/audio-backend — плейер ожидает интеграции
  после выбора аудио-стека в PySide6 (QMediaPlayer или pygame).
- Интеграция установки Torch/RVC/llama.cpp по кнопке на вкладке «Окружение» —
  функции `install_torch()`, `install_rvc()`, `install_llama()` в `env_core`
  уже есть, кнопки готовы, нужно подключить их в фоновом worker.

Эти интеграции — это уже «подключение сигналов к готовым функциям ядра», а не
архитектурные доработки. Каркас GUI готов, все 9 панелей рендерятся и базовые
элементы (нормализация, превью чанков, навигация, статус зависимостей, диагностика,
пресеты, история, словарь замен) работают уже сейчас.
